## 검증 결과 (샌드박스 실행)

**`python -m engine selftest`** — 합성 device CMYK 6사이즈×4조각=24배치
→ 10개 검사 전부 **PASS**, Ghostscript inkcov 편차 **0.000000**. (Phase 0 완료기준 "engine 단독 실행으로 PoC 재현" 충족)

**`python -m engine build`** — 실제 운영 파일
`design_XL.ai`(%PDF-1.4, ICCBased CMYK, 4478×5669pt) + `pattern_XS.svg`(폴리라인 3조각, viewBox 4337×3401)
→ U넥 곡선 패턴 윤곽으로 농구유니폼(YONSEI/김경원/#20) 디자인을 **무손실 클리핑 렌더 성공**(미리보기 확인).
→ 검증: 무손실보존·단일임베드·CMYK유지·합성정확 **PASS** / 투명도만 **FAIL**.

## 핵심 발견

1. **검증을 바이트 동일성 기반으로 교체**: PoC의 k/K 색값 추출은 합성 device CMYK 전용.
   실제 디자인은 **ICCBased CMYK**라 k/K가 0개로 잡혀 오탐. → 임베드 Form 콘텐츠가 원본과
   **byte-identical**인지로 검사하면 색공간 종류와 무관하게 무손실을 증명한다(동일 사본 1개 = 단일 임베드도 동시 증명).

2. **투명도 FAIL = 안전장치 정상**: 테스트 디자인의 20% 워터마크(부엉이 로고·방패)를 정확히 검출.
   이건 버그가 아니라 "공장 전달 전 차단"이 작동한 것. → 원본에서 1회 평탄화 후 재등록이 운영 규칙.

3. **디자인↔패턴 좌표 정합이 다음 핵심**: 디자인 페이지 좌표계 ≠ 패턴 viewBox 좌표계.
   "디자인의 어느 영역이 어느 조각에 얹히는가"는 자동추정 불가 → **패턴 프리셋(preset.json)에
   조각별 디자인 매핑(좌표/스케일)을 정의**해야 한다(기존엔 일러 레이어가 담당).
   현재 `build`는 데모로 전체 디자인을 각 조각 bbox에 fit.

## 다음 단계 (우선순위)
