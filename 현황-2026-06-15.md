# grader-v2 통합 현황 — 2026-06-15 (코워크 허브 정리)

> 코워크(허브)가 **디자인 클로드 산출물**과 **CLI 구현 진척**을 함께 검토하고 다음 작업을 정렬한 문서.

## 1. 두 갈래 진행 상황

### A. 디자인 클로드 (UI/UX) — ✅ 1차 납품 완료
- `STIZ grader Design System.zip` 수령 → 프로젝트 `web/`에 통합.
- 구성: 토큰(색/타이포/간격) + 화면 3개(작업·패턴관리·작업기록) + 문서 2개(flow·states) + STIZ 로고. 순수 HTML/CSS/vanilla JS, CDN 2개(Pretendard·Material Symbols)만 의존.
- 품질: **의뢰서 그대로 충족.** 5단계 스텝퍼·점진적 노출·투명도 경고 차단·추출 실패 셀 강조·검증 배지(전달가능/확인)·패턴관리 드래그 영역 에디터까지 실제 인터랙션 구현. API 계약 mock 데이터 형태도 의뢰서 7번과 일치.
- **코워크가 수정한 점**: 출력 포맷 EPS→PDF 결정이 카피 5곳에 미반영 → `web/`에서 PDF로 정정(STEPS 변수 보호). 원본 zip은 보존.

### B. CLI 구현 — ✅ Phase A-1 완료 (좌표정합 해결)
- git: `A-1 좌표정합 방식 A(앵커 정합) 확정` → `A-1 완료 — preset.json 기반 grade.py + cli grade`.
- **최우선 난제였던 "디자인↔패턴 좌표 정합"을 해결**: 방식 A(앵커 bottom-left + contain) 채택, 방식 B(전체정렬+클립)와 시험렌더 비교 후 확정(`grading_compare/`).
- 신규: `engine/grade.py`, `data/patterns/농구_U넥_양면/preset.json`(content_box·조각별 design_region_pt·앵커 매핑·배번/이름 rel_bbox·shrink 자리).
- 결과: `_grade_out/grade_p01.png` — 앞판(YONSEI 20)·뒤판(김경원 20)이 실제 U넥 패턴 조각에 정확 정렬. verify는 투명도 외 전 항목 PASS(테스트 디자인 워터마크 때문, 안전장치 정상).

## 3. 다음 작업 우선순위 (제안)

**병렬 2트랙으로 진행 가능** (A-1·디자인이 끝나 서로 막힘 없음):

**트랙 1 — 엔진 완성 (CLI, UI 불필요):**
1. **A-4 배번·이름 `text.py`** — fontTools 글리프→벡터 경로, preset의 number/name_area에 CMYK 배치. (실제 자동화 가치가 큰 핵심 기능)
2. **A-5 주문서 `order.py`** — xlsx→행 추출(기존 order_parser 지식 이식).
3. **A-2 전 사이즈** — XS~5XL 패턴 확보 후 per-size 매핑. (실제 사이즈별 패턴 파일 필요 → 사용자/공장 협조)

**트랙 2 — 웹앱 골격 (CLI, 디자인 결합):**
4. **Phase B FastAPI** — `engine` import, API 계약 엔드포인트 구현, `web/` 정적 UI 마운트 + mock→실데이터 치환.

## 4. 사용자(수빈) 확인/준비 필요
- **전 사이즈 패턴 파일**(XS~5XL SVG/DXF) — A-2의 입력. 현재 테스트는 XS 1개뿐.
- **폰트 파일**(배번/이름용 ttf/otf) — A-4 입력. 라이선스 확인.
- **공장 포맷 확정** — 사이즈별 개별 PDF vs 다페이지 1 PDF.
- **공장 시험인쇄** — Phase A 완료의 절대 조건(미리보기 통과 ≠ RIP 통과).
