# 의뢰서 (디자인 Claude 용) — STIZ grader UI 시안 보강

> 대상: `STIZ grader` UI 킷(디자인 시스템 + ui_kits/grader/ 화면들)
> 목적: 실제 엔진(FastAPI 연결) 붙이기 전에, **운영에서 실제로 터진 이슈**를 화면에 반영.
> 원칙: 기존 디자인 시스템(컴포넌트·토큰·브랜드)을 **그대로 재사용**. 새 컴포넌트는 최소화.
> 비고: 백엔드는 별도(엔진 run_job/parse_order/flatten/verify/preview). 이 문서는 **화면/UX만**.

## 0. 지금 시안 평가 — 잘 돼 있음(유지)
3화면(작업/패턴 관리/작업 기록), 작업 5단계(패턴→디자인→주문서→생성→검수), 투명도 경고(진행
차단), PostScript-AI 거부 경고, CMYK 점검, 주문표 인라인 편집, 진행바, 검수 그리드+검증 배지,
ZIP, 패턴 등록(사이즈별 업로드 + 배번/이름 영역 드래그). **이 골격은 바꾸지 말 것.** 아래는
"빠진 상태/단계"를 같은 톤으로 끼워넣는 작업이다.

---

## 1. [작업 → 디자인 단계] 업로드 점검 상태 1개 추가 — "본체 누락"
**왜**: 운영 중 가장 크게 데인 버그. 파일이 PDF 형식은 맞는데(=PostScript 거부에 안 걸림)
**디자인 본체가 PDF에 안 들어간**(일러스트 전용 데이터에만 있고 PDF 콘텐츠는 거의 빔) 경우가
있었다. 이러면 출력이 통째로 흰 화면이 된다. 투명도/형식과는 **다른** 점검 항목.
**요청**: 디자인 업로드 직후 점검 상태에 **3번째 실패 케이스** 추가(기존 "투명도 경고",
"PDF 호환 아님"과 나란히, `alert--warn`/`filecard--warn` 재사용):
- 제목: "디자인 본체가 PDF에 포함되지 않았습니다"
- 본문: "일러스트레이터에서 **‘PDF 호환 파일 만들기’를 켜고 다시 저장**하면 본체가 포함됩니다.
  (파일은 열리지만 그림이 비어 있는 상태예요.)" + 진행 차단.
- 통과 케이스 파일카드 메타에 "본체 포함 ✓" 한 줄 추가(예: `CMYK · 투명도 없음 · 본체 포함 · 2.4MB`).

## 2. [작업 → 디자인 단계] 슬롯 라벨 명확화 — "빈 본체"
**왜**: '완성본(샘플 번호·이름 박힌 것)'과 '빈 본체(번호·이름 없는 템플릿)'를 혼동해 오래 헤맴.
**요청**: 드롭존 제목/힌트를 "**빈 본체 템플릿**(번호·이름 없는 기준 사이즈 1개)"으로 명시.
업로드된 파일에 베이크된 번호/이름이 감지되면 경고 노트("이 파일엔 번호/이름이 이미 들어 있어요 —
빈 본체를 올려주세요"). 보조 설명 1줄: "완성본(샘플)은 패턴 등록의 ‘기준’으로만 쓰입니다."

## 3. [작업 → 검수 단계] 검증 항목 2개 보강
**왜**: 공장 재단 필수 요소가 검수에 안 보임 + 안전 차단 결과가 안 보임.
**요청**:
- 검증 체크리스트(현재 무손실/단일임베드/CMYK/투명도/합성정확)에 **"재단선·너치 보존"** 1줄 추가
  (CheckList 그대로). detail 예: "조각 외곽 재단선 3 + 너치 N".
- 검수 상단(또는 요약 레일)에 **"건너뜀 N건" 알림** 영역 추가: 사이즈 누락·패턴 결함으로 제외된
  주문 행을 사유와 함께 표시(예: "3XL 5건 — 패턴 파일 결함으로 제외"). `Badge tone="warn"` + 목록.

## 4. [패턴 관리 → 등록] 입력 2가지 추가
현재 등록은 "사이즈별 패턴 파일 + 배번/이름 영역 드래그"만 받는다. 아래 2개가 빠졌다.
**4-A. 번호 글리프셋(0~9 아웃라인) 등록**
- **왜**: 실제 유니폼 번호 폰트 = 디자이너가 아웃라인한 0~9(별도 아트보드/파일). 시스템 폰트가
  아님. 이게 없으면 번호 모양이 달라진다.
- **요청**: 등록 화면에 "번호 글리프셋" 업로드 슬롯 1개(Dropzone, accept: "0~9 아웃라인 AI/PDF/SVG").
  업로드되면 0~9 미리보기 한 줄 + "10자 인식됨 ✓". 없으면 "기본 폰트로 대체" 안내(폴백 허용).
**4-B. 완성본 기준 업로드 → 영역 자동 추출(권장 경로)**
- **왜**: 배번/이름의 **위치·크기·자간**은 수동 드래그로 정확히 못 맞춘다. 완성본 1장을 올리면
  자동 추출하는 게 정답(엔진 지원). 드래그는 미세조정/폴백용.
- **요청**: 영역 지정 섹션 상단에 탭/토글 2개 — **"완성본에서 자동 추출(권장)"** vs "직접 드래그".
  자동 추출 선택 시: 완성본 Dropzone + 추출 결과 카드("배번 중심·높이 / 이름 위치·자간 자동 인식됨",
  값 수정 가능 input들). 드래그는 기존 RegionBox UI 유지.

## 5. [신규 화면] 설정(Settings)
**왜**: 실제 구동엔 경로·색기준·서버 설정이 필요(구버전 grader엔 Drive 설정 있었음). 현재 없음.
**요청**: 사이드바에 "설정" 항목 추가 + 화면 신설(기존 forms 컴포넌트 Input/Select/Switch 재사용):
- 경로: 패턴 폴더 / 디자인 폴더 / 폰트 폴더 / 출력 폴더 (각 Input + 폴더선택 버튼).
- 출력: 형식(EPS / PDF, Select), per_player·single(Switch/Select), 미리보기 PNG 생성(Switch).
- 색: "실제 출력색은 공장 기준" 고지 + 색프로파일 메모.
- 서버: 연결 상태(이미 사이드바 하단에 있음) 상세.

## 6. [작업 → 생성/검수] 출력 형식 선택 (PDF / EPS / 둘 다)
**확정**: 출력 형식은 **PDF·EPS 둘 다 지원**하고, **필요한 걸 선택**해서 뽑는다(둘 다도 가능).
- 설정의 출력 형식 Select 옵션을 **`PDF` / `EPS` / `PDF+EPS(둘 다)`** 로(기존 vector 중립은 기본값 유지 가능).
- 단순 설정 기본값에 더해, **작업 흐름에서도 선택**할 수 있게: 생성 단계(또는 검수 다운로드 직전)에
  형식 선택 컨트롤(세그먼트/Select) 1개 — 주문마다 형식을 바꿀 수 있어야 함.
- 표기 연동: 진행바 "{형식} 생성 중…", 검수 다운로드 "ZIP 다운로드({형식})". 둘 다일 때 ZIP 안은
  `pdf/`·`eps/` 하위폴더로 분리됨을 안내(검수 화면 한 줄).
- (참고) EPS는 벡터 유지를 위해 평탄화가 선결이며 엔진이 자동 처리 — 화면은 형식 선택만 노출하면 됨.

## 7. 디자인 시스템 제약 (반드시)
- 기존 토큰(`tokens/*.css`)·컴포넌트(core/forms/feedback/navigation/work)만 사용. 새 원자
  컴포넌트 추가 금지(조합으로 해결). 브랜드(앰버/블랙/뉴트럴) 유지.
- 추가 화면/상태도 `static/`(정적 핸드오프 HTML)과 `ui_kits/grader/`(React 목업) **양쪽에** 반영,
  `data.js` 목 데이터에 새 필드(글리프셋 유무, skip 목록, 본체검증 상태, 설정값) 추가.
- 카피 톤: 기존처럼 "친절하지만 단호하게 진행 차단"(비개발자 직원 대상, 한국어).

## 8. 산출물
- 업데이트된 ui_kits/grader 화면(작업·패턴·기록 + 신규 설정) + static/ 핸드오프 + data.js.
- 변경 요약(어느 화면에 무엇을 추가했는지) 1장.

## 9. 우선순위
1·2·3 (작업 플로우 핵심·운영 사고 직결) → 4 (패턴 등록, 신규 디자인마다 필요) →
5·6 (설정·형식, 구동 편의). 1~4가 핵심.
