# 농구_U넥_양면 패턴 프리셋 설명

이 폴더는 "농구 U넥 양면" 패턴 1종의 **그레이딩(전 사이즈 자동 합성) 설계도**입니다.
preset.json 은 순수 JSON이라 파일 안에 주석을 못 답니다. 그래서 각 항목 뜻을 여기에 한글로 정리합니다.

비유: preset.json 은 "요리 레시피 카드"입니다. 디자인 1장(기준 사이즈)을
패턴 조각들 위에 "어디에, 어떤 크기로 얹어라"를 적어둔 표입니다.

---

## 폴더 구성
- `preset.json` : 설계도 본체 (아래 항목 설명 참고)
- `XS.svg` : XS 사이즈 패턴 윤곽선 파일 (조각들의 외곽선이 들어있음)
  - 원본 위치: `grader/illustrator-scripts/test/pattern_XS.svg` 를 이 폴더로 복사한 것.
  - preset.json 의 `pattern_file` 은 **이 폴더 기준 상대경로**로 가리킵니다(외부 절대경로 의존 제거).
- `README.md` : 이 설명 파일

> 사이즈가 늘어나면 `S.svg`, `M.svg`, `L.svg` … 를 이 폴더에 넣고
> preset.json 의 `sizes` 배열에 한 줄씩 추가하면 됩니다.

---

## preset.json 항목별 설명

### preset_name / version
- `preset_name` : 패턴 이름. 폴더 이름과 같게 둡니다.
- `version` : 스키마 버전. 구조가 바뀌면 숫자를 올립니다(지금은 1).

### design (기준 디자인 정보)
- `base_size` : 기준 디자인이 어느 사이즈인지. 여기선 **XL**.
  (디자이너가 XL 1장만 그리면, 나머지 사이즈는 자동 합성됩니다.)
- `design_file` : 기준 디자인 파일 이름(참고용 메모). 실제 합성 시엔 호출하는 쪽에서 경로를 넘깁니다.
- `page_size_pt` : 디자인 페이지(도화지) 크기. 단위는 pt(포인트). `[가로, 세로]`.
- `content_box` : 디자인 안에서 실제 그림이 들어찬 영역(메모용). `[x최소, y최소, x최대, y최대]`.

### sizes (만들 사이즈 목록)
배열입니다. 한 칸이 사이즈 1개 = 출력 PDF 1페이지가 됩니다.
- `name` : 사이즈 이름(XS, S, M …).
- `pattern_file` : 그 사이즈 패턴 SVG 파일 이름(이 폴더 기준 상대경로).
- `scale` : 보조 배율(기본 1.0). 같은 SVG를 더 키우거나 줄여야 할 때 쓰는 자리.
  보통 사이즈마다 SVG 파일 자체가 다르므로 1.0 그대로 둡니다.

> 지금은 **XS 1개만 실제 등록**되어 있습니다.
> 나머지 사이즈는 SVG 파일을 넣고 sizes 에 줄을 추가하면 바로 확장됩니다(코드 수정 불필요).

### pieces (조각 정의) — 그레이딩의 핵심
조각(앞판/뒤판/소매)마다 "디자인의 어느 부분을 가져올지"를 적습니다.
- `id` : 조각 식별자(영문). front/back/sleeve.
- `name` : 사람이 읽는 이름(앞판/뒤판/소매).
- `svg_index` : 패턴 SVG에서 이 조각이 **몇 번째 조각인지**.
  - 주의: 패턴 파서(parse_svg)는 조각을 **높이가 큰 순서**로 정렬합니다.
  - 그래서 `0=뒤판(가장 큼), 1=앞판, 2=소매(가장 작음)` 입니다. (실측 확정)
- `design_region_pt` : 디자인에서 이 조각이 차지하는 직사각형 영역.
  `[x최소, y최소, x최대, y최대]` (pt, 좌하단이 원점).
  - 앞판: `[468, 2877, 2098, 5287]`
  - 뒤판: `[2310, 2877, 3997, 5287]`
  - 소매: `[468, 5301, 2225, 5562]`

### design_mapping (얹는 방식) — "방식 A: 앵커 정합"
디자인 영역을 패턴 조각 위에 얹는 규칙입니다.
- `mode` : `"anchor"` (앵커 기준 정합). 확정된 방식.
- `anchor` : 정렬 기준점. `"bottom-left"` = 디자인 영역의 좌하단을 조각 윤곽의 좌하단에 맞춥니다.
- `fit` : `"contain"` = 디자인 영역이 조각 밖으로 삐져나오지 않도록,
  가로/세로 배율 중 **작은 쪽**을 써서 통째로 안에 담습니다.
- `preserve_aspect` : `true` = 가로세로 비율을 깨지 않음(찌그러뜨리지 않음).
  엔진의 스케일이 등방(가로=세로 같은 배율)이라 항상 true 입니다.

### number_area / name_area (배번·이름 자리) — 좌표만 정의 (글자 렌더는 A-4)
배번호와 이름이 들어갈 자리를, **조각 bbox 안에서의 비율(0~1)**로 적습니다.
비율을 쓰는 이유: 사이즈가 커지면 조각도 커지므로, 절대좌표보다 비율이 자동으로 따라옵니다.
- `piece_id` : 어느 조각에 넣을지(보통 `back` = 뒤판).
- `rel_bbox` : `[x시작, y시작, x끝, y끝]` 비율.
  - 0 = 조각 왼쪽/아래 끝, 1 = 조각 오른쪽/위 끝.
  - number_area : 뒤판 중앙쯤(배번호).
  - name_area : 그 배번호 **위쪽**(이름). y값이 더 큼 = 더 위.
- 지금은 **자리(좌표)만** 정의한 상태입니다. 실제 글자 그리기는 A-4 단계에서 합니다.

### shrink (수축 보정) — 자리만 확보
- `x`, `y` : 인쇄 후 천이 줄어드는 비율을 미리 보정하는 계수. 지금은 둘 다 1.0(보정 없음).
- 나중에 공장 데이터가 나오면 예: `x:0.98, y:0.97` 처럼 넣어 미리 늘려 출력합니다.
